# Характеристика проекта

Проект был поэтапно доработан в рамках практических заданий №7–11 по дисциплине «Фронтенд и бэкенд разработка».  
В результате реализованы:

- REST API для пользователей и товаров
- регистрация и вход пользователя
- хеширование паролей через `bcrypt`
- выдача `access token` и `refresh token`
- защищённые маршруты через JWT
- endpoint для получения текущего пользователя
- автоматическое обновление access token на клиенте
- React-интерфейс с формами регистрации и входа
- каталог товаров, просмотр карточки товара по `id`
- создание, редактирование и удаление товаров
- Swagger-документация API

---

## Функциональность

### Пользователи
- регистрация нового пользователя;
- вход в систему по `email` и `password`;
- хеширование паролей перед сохранением;
- получение данных текущего пользователя через защищённый маршрут `/api/auth/me`;
- выход из системы;
- обновление access token через refresh token.

### Товары
- получение списка всех товаров;
- получение товара по `id`;
- создание нового товара;
- редактирование товара;
- удаление товара;
- просмотр детальной страницы товара.

### Клиентская часть
- страница входа;
- страница регистрации;
- страница каталога товаров;
- страница просмотра товара по `id`;
- защищённые маршруты;
- хранение токенов в `localStorage`;
- автоматический `refresh` токена после ответа `401 Unauthorized`.

## Что было реализовано

### Практическая работа №7
В проект добавлены базовые backend-маршруты:

- `POST /api/auth/register`
- `POST /api/auth/login`
- `POST /api/products`
- `GET /api/products`
- `GET /api/products/:id`
- `PUT /api/products/:id`
- `DELETE /api/products/:id`

Также:
- добавлена Swagger-документация;
- реализована валидация данных;
- добавлено хеширование паролей через `bcrypt`.

### Практическая работа №8
Проект расширен JWT-аутентификацией:

- сервер выдаёт `accessToken` при логине;
- добавлен защищённый маршрут `GET /api/auth/me`;
- защищены маршруты просмотра товара по `id`, обновления и удаления.

### Практическая работа №9
Добавлены refresh token и ротация токенов:

- реализован маршрут `POST /api/auth/refresh`;
- refresh token передаётся в заголовке;
- при обновлении токена сервер удаляет старый refresh token и создаёт новую пару токенов;
- добавлен `POST /api/auth/logout`.

### Практическая работа №10-11
Реализована полноценная клиентская часть:

- отдельные страницы входа и регистрации;
- каталог товаров;
- детальная страница товара;
- формы создания и редактирования товара;
- автоматическое обновление токенов через `axios.interceptors`;
- клиентская маршрутизация через `react-router-dom`.

## API

### Auth
- `POST /api/auth/register` — регистрация пользователя
- `POST /api/auth/login` — вход в систему
- `POST /api/auth/refresh` — обновление пары токенов
- `POST /api/auth/logout` — выход из системы
- `GET /api/auth/me` — получить текущего пользователя

### Products
- `GET /api/products` — получить список товаров
- `POST /api/products` — создать товар
- `GET /api/products/:id` — получить товар по `id`
- `PUT /api/products/:id` — обновить товар
- `DELETE /api/products/:id` — удалить товар
